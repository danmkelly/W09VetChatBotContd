The Business Challenge

Bringing the vet clinic counter online

Meadow Vet Care is a modern Irish veterinary practice serving dogs, cats, rabbits, small mammals, and birds. With 90+ services across consultations, preventive care, vaccinations, dentistry, surgery, diagnostics, grooming, and behaviour — their offering is comprehensive but complex for customers to navigate.

The problem: customers call the clinic to ask the same questions: What dog services do you offer? Are there any offers on microchipping? Do you have telehealth services? What's the price for a cat dental? Is there availability this week? These repetitive queries tie up reception staff and frustrate customers who just want quick answers.

The answer: a friendly, AI-powered chatbot named MeadowBot that knows every service, every price, every special offer, and every available slot — straight from the clinic's live data.

The MCP Concept

Giving an AI a live tool: the Model Context Protocol pattern

This week's module concept is MCP — Model Context Protocol: giving an LLM access to a live data source so it can answer questions with real, up-to-date information rather than relying on training data alone.

For Meadow Vet Care, the live data source is a Google Sheet containing the clinic's entire service catalogue: 94 services, each with a service ID, category, species, price, duration, appointment requirements, availability window, slots remaining this week, and any active special offers.

When a customer asks "Are there any offers on microchipping?", MeadowBot fetches the live sheet, filters for Microchip & ID category services with non-empty special offers, and replies with the exact prices and offer details. No hard-coded data. No stale information. One source of truth.

How MeadowBot Works

AI-powered vet clinic assistant, grounded in live data

🤖

Conversational AI

MeadowBot is powered by DeepSeek, a large language model. She understands natural questions like "How much is a dental scale and polish for my medium-sized dog?" and responds with accurate, friendly advice drawn from the live service sheet.

📊

Live Google Sheet (MCP)

All 94 services, prices, offers, and slot availability are fetched live from the clinic's Google Sheet via CSV export. The chatbot queries the sheet in real time — no stale data, no hard-coded catalogue. This is the MCP pattern in practice.

📅

Booking & Calendar

Services that require appointments surface an inline calendar booking flow. Customers pick a day, see available time slots, and submit a booking request with their pet's details. A simple, friction-free path from question to booking.

Limitations & Known Weak Spots

Where MeadowBot still struggles

Like any real-world AI project, MeadowBot has rough edges. Here are the honest ones:

🔎

NLP Intent Parsing

MeadowBot uses keyword matching + LLM disambiguation to understand whether a user is asking about services, prices, offers, or booking. Complex compound queries ("how much for a dental for my cat and is there an offer on vaccinations for my dog?") can confuse the intent router. The slot-filling gate asks clarifying questions when confidence is low, but multi-pet, multi-service queries remain challenging.

📊

Sheet Data Latency

The Google Sheet is fetched at page load and cached for the session. Slot counts and offers update only on refresh. If the clinic updates the sheet during a customer session, the chatbot won't reflect those changes until the page reloads. For a production system, a server-side cache with TTL-based refresh would be needed.

⚠

Conversation Context

MeadowBot maintains conversation history (last 10 exchanges) within the current session, and persists memory across visits using browser localStorage — including your name, your pets' names, and recent service searches. On return visits, MeadowBot greets you personally and references past queries. This memory is device-only: no data leaves your browser. A "Clear memory" button lets you wipe it at any time.

📅

Booking Is Simulated

The calendar booking flow is a front-end simulation: it captures the selected service, date, time, and pet details into localStorage, but does not write back to any real booking system. This is appropriate for the academic scope but would need a real API integration (e.g. Google Calendar, Cliniko, or Provet Cloud) for production use.

🎤

Voice Input Limitations

Voice recognition uses the browser's built-in Web Speech API, which varies in accuracy across browsers and devices. Pet breed names, medical terms, and Irish addresses are particularly prone to transcription errors. Voice output (TTS) may struggle with veterinary terminology.

💻

No Real Backend

MeadowBot is a pure client-side application with no server-side database, authentication, or booking persistence. The Cloudflare Worker proxy handles API key security, but all other state is browser-local. This is deliberate for the academic scope but limits production-readiness.

Technical Architecture

Secure, serverless, and key-safe

The LLM API (DeepSeek) requires a secret key. This is never exposed to the browser. A Cloudflare Worker acts as a thin proxy layer: the browser calls the worker, the worker injects the secret key server-side, and the response flows back. The Google Sheet data is fetched client-side via public CSV export (no auth needed for reading). No keys in source code, no keys in DevTools.

Development Approach

Built by a team of AI agent colleagues + MCP tools

The project was developed using a structured team of specialist AI agents, each with a defined role, domain expertise, and boundaries. This mirrors how a real innovation team would operate.

The team's design decisions are grounded in four academic best-practice frameworks from the Customer Engagement & AI module: the seven-dimension chatbot taxonomy (Adamopoulou & Moussiades 2020), CASA / Computers Are Social Actors (Reeves & Nass 1996), conversation flow anatomy, and Grice's maxims of conversation (Grice 1975). Each specialist agent owns the framework dimensions relevant to their domain.
